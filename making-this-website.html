Making this website

Ben Hutchins April 6, 2026

Hi, this is my first foray into writing content for the web, using more than just a code editor, and github pages. Now it includes Publii, which is what Matt Sayar used.

I hope to write about interesting things, especially on the topic of infrastructure (or as it used to be known, public works).

Here's how you can do it too:

1. Download and install Publii

2. I already had a repo called hutcho.github.io so I wanted to use this. It hosted a static site, with very minimal content. I rewrote it into Publii pages.

3. Publii server settings. This was hard to figure out, because the hints inside publii are wrong as of 2026-04-07:

Website URL: https://hutcho.github.io           

This was the most confusing. The hint says YOUR_USERNAME.github.io/YOUR_REPOSITORY_NAME. But this is wrong. It should just be YOUR_USERNAME.github.io

API server (leave as it): api.github.com

Username: hutcho

Repository: hutcho.github.io

Branch: main

Token: Generate this yourself on github. I tried using the newer style tokens, but after a few attemps gave up and used classic style. Copied from Publii docs and did what they say to do for the token permissions.

Parallel uploads: I changed this to 3.

API rate limiting: On (blue).
